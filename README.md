
# Lamatic Test Frontend

## 1. How to Install and Run the Project

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/SakshamGoel15/lamatic-test-frontend.git
   ```
2. Navigate to the project directory:
   ```bash
   cd lamatic-test-frontend
   ```
3. Install dependencies:
   ```bash
   yarn
   ```

### Running the Client

- Provide commands to run the server:
  ```bash
  yarn dev

- To run the client locally, access it at http://localhost:3030/. For remote access, the client is also deployed on Netlify.

## 3. Tech Used for Frontend

- **List of the technologies and frameworks used in the frontend:**
  - **React**: A library for building user interfaces using components.
  - **Material-UI (MUI)**: A UI component library based on Material Design for React applications.
  - **Redux**: A state management library used for managing application state.
  - **React Router**: A library for routing and navigation in React applications.

## 4. Features of the Application

- `Personalized Sign-In`: Users can sign in using their name, offering a personalized experience right from the start.
- `Interactive Q&A with LLMs`: It allows users to pose questions and receive responses from various large language models (LLMs), tapping into a wealth of knowledge and perspectives.
- `Multiple LLM Selection`: Users have the flexibility to select different LLMs simultaneously for a broader range of insights.
- `Session-based Multitasking`: Within a single session, users can inquire about diverse topics, facilitating a seamless flow of information.
- `Dynamic Data Processing`: The application intelligently processes mentions of "Saksham" by referencing "Aman," showcasing its capability to dynamically alter content based on pre-set rules.
- `Voice Interaction`: Incorporating speech-to-text for query input and text-to-speech for auditory response playback, this feature ensures accessibility and convenience, enhancing the overall user experience.
